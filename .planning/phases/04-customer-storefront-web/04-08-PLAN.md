<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API responses → rendered UI | Product/block/category data rendered into the DOM |
| URL params → query | User-controlled filter/sort/query strings drive API calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-23 | Tampering | URL filter params | mitigate | Filter params are forwarded to the Phase 3 search API which validates them; the client encodes values (encodeURIComponent) and does not eval URL content |
| T-04-24 | Information Disclosure | block payload rendering | mitigate | Block content is rendered as text/JSX (React escapes by default); no dangerouslySetInnerHTML for untrusted block content |
| T-04-25 | Tampering | disabled Add to Cart | accept | Button is inert (`disabled` + no handler that mutates); commerce authority arrives in Phase 5 — no client-side state to tamper with in Phase 4 |
</threat_model>
