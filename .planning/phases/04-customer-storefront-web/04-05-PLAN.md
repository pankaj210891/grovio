    `apps/api/src/routes/customer/auth.ts` → `customerAuthRoutes` (public, NO auth guard):
    - POST `/auth/signup`: RegisterInputSchema `{ email: z.string().email(), password: z.string().min(8), name: z.string().min(1) }`; calls register(); 201 on success; 409 with err.code on CustomerAlreadyExistsError.
    - POST `/auth/login`: LoginInputSchema `{ email: z.string().email(), password: z.string().min(1) }`; calls login(); on success sets `access_token` cookie `{ httpOnly: true, secure: NODE_ENV==="production", sameSite: "lax", path: "/", maxAge: expiresIn }` and `refresh_token` cookie `{ httpOnly: true, secure, sameSite: "lax", path: "/auth/refresh", maxAge: 604800 }` (refresh scoped path per Pitfall 7), responds `{ success: true, data: { expiresIn } }`; 401 with err.code on InvalidCredentialsError.
    - POST `/auth/refresh`: reads `request.cookies.refresh_token`, calls refreshTokens(), re-sets the access_token (and refresh_token) cookies; 401 if missing/invalid.
    - POST `/auth/logout`: clears both cookies via `reply.clearCookie("access_token", { path: "/" })` and `reply.clearCookie("refresh_token", { path: "/auth/refresh" })`; returns 200. (Logout is reachable without a strict guard but only clears cookies.)
    - POST `/auth/forgot-password`: `{ email }`; calls forgotPassword(); always returns 200 `{ success: true }` (no enumeration).
    - POST `/auth/reset-password`: `{ token: z.string(), password: z.string().min(8) }`; calls resetPassword(); 200 on success; 400 with err.code on InvalidResetTokenError.

    `apps/api/src/routes/account/profile.ts` → `accountProfileRoutes`: `fastify.addHook("preHandler", requireCustomerAuth)` first. GET `/account/profile` returns the customer's name/email/phone (resolved via customer-auth or a small query using `request.customerId`). PATCH `/account/profile` with `{ name?, phone? }` updates and returns the customer. 401 handled by middleware.

    `apps/api/src/routes/account/addresses.ts` → `accountAddressRoutes`: `preHandler` requireCustomerAuth. GET `/account/addresses` → listAddresses(request.customerId). POST `/account/addresses` with the structured address body (street/city/state/pincode/country required; label/lat/lng/placeId/isDefault optional) → createAddress(request.customerId, body). PATCH `/account/addresses/:id` → updateAddress(id, request.customerId, body); 404 if null. DELETE `/account/addresses/:id` → deleteAddress(id, request.customerId); 404 if nothing deleted.

    `apps/api/src/routes/homepage.ts` → `homepageRoutes` (public): GET `/homepage` → `homepageService.getBlocks()` → `{ success: true, data: { blocks } }`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| storefront origin → API | Cross-origin requests carry credentials (cookies); CORS must allow exactly one origin |
| client → /account/* | Customer cookie validated before handler runs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-13 | Information Disclosure | CORS with credentials | mitigate | `origin: env.STOREFRONT_ORIGIN` (specific, never `*`) + `credentials: true`; wildcard+credentials combination avoided (Pitfall 2) |
| T-04-14 | Information Disclosure | refresh token surface | mitigate | refresh_token cookie scoped to `path: "/auth/refresh"` so it is not sent to other endpoints (Pitfall 7) |
| T-04-15 | Spoofing | /account/* access | mitigate | requireCustomerAuth preHandler validates the access_token cookie + customer role before any handler runs |
| T-04-16 | Information Disclosure | stale token after logout | mitigate | logout clears both cookies with matching path/sameSite attributes |
| T-04-17 | Tampering | request bodies | mitigate | Inline Zod schemas validate every route body before the service is called (V5 input validation) |
</threat_model>
