<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser form → /auth + /account | User credentials and address data submitted from the client |
| browser → Google Maps CDN | Places autocomplete loads third-party script with the Maps API key |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-20 | Information Disclosure | auth error copy | mitigate | Login shows a single generic "Incorrect email or password" regardless of which field was wrong (no enumeration) — server already enforces this |
| T-04-21 | Tampering | reset token in URL | mitigate | Token is single-use and server-validated; client only forwards it to POST /auth/reset-password (D-10) |
| T-04-22 | Information Disclosure | Google Maps API key in client | accept | Maps JS API keys are public by design; mitigation is HTTP-referrer restriction configured in the Google Cloud console (documented for the buyer), not a code concern |
</threat_model>
