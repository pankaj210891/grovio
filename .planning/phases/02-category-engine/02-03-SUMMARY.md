- `CREATE TABLE "vendor_category_restrictions"` — categoryId cascade FK, vendorId/createdByAdminId as bare UUIDs (deferred FKs per T-02-05), composite unique (category_id, vendor_id)
- `CREATE TABLE "category_metadata"` — unique categoryId FK, flat SEO columns, JSONB blocks default []
- Drizzle meta journal (`_journal.json`) and snapshot (`0000_snapshot.json`) for migration tracking

### CategoryService (`apps/api/src/modules/categories/CategoryService.ts`)

Key public methods:
- `getTree()` — Redis-first: redis.get("cat:tree") → JSON.parse on hit; db.select.where(isNull(archivedAt)).orderBy(asc(sortOrder)) → buildTree → redis.setex on miss
- `getCategoryById(id)` — direct DB read, no caching
- `createCategory(input)` — depth guard, slug resolution, insert, invalidateTree
- `updateCategory(id, input)` — partial update, invalidateTree
- `archiveCategory(id)` — sets archivedAt = now, invalidateTree
- `reorderCategories(parentId, orderedIds)` — batch sort_order = index, invalidateTree
- `invalidateTree()` — redis.del("cat:tree")

Private helpers:
- `buildTree(rows)` — O(n) two-pass adjacency list → nested CategoryTreeNode[]
- `getDepth(categoryId)` — walks parent chain via sequential DB selects
- `slugify(source)` — URL-safe: lowercase, replace non-alnum with hyphens
- `resolveSlug(source, excludeId?)` — slugify + uniqueness check + -2/-3 collision suffix
