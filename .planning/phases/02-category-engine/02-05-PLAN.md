|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Tampering | malformed JSONB blocks injection | mitigate | CategoryMetadataService runs MerchandisingBlockSchema.array().parse() before any DB write; malformed/unknown blocks are rejected with a 400 instead of corrupting JSONB and breaking Phase 4 rendering (Pitfall 5) |
| T-02-13 | Tampering | orphan vendor_ids without FK | accept | vendor_id has no FK until Phase 3; admin restriction routes are dev-only/guarded (plan 02-06) so the table is empty at Phase 2 completion (Pitfall 6, D-10) |
| T-02-14 | Tampering | duplicate vendor restriction | mitigate | unique(category_id, vendor_id) DB constraint + service-level clean error |
| T-02-SC | Tampering | npm installs | accept | No package installs — deps already in apps/api; legitimacy gate handled in plan 02-07 |
</threat_model>
