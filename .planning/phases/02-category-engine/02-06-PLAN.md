## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → public /categories routes | Untrusted read traffic; no mutation, cache-backed |
| admin client → /admin/categories routes | Privileged mutation surface; must not be publicly accessible before Phase 4 auth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-15 | Elevation of Privilege | unauthenticated admin category mutation | mitigate | Placeholder guard (NODE_ENV !== production OR X-Internal-Admin-Token header) on the admin route plugin; documented Phase 4 JWT replacement (Pitfall 8, ASVS V4) |
| T-02-16 | Tampering | malformed admin request bodies | mitigate | Every admin route parses its body through the corresponding `@grovio/contracts` Zod schema before reaching the service (ASVS V5) |
| T-02-17 | Information Disclosure | raw error leakage on domain failure | mitigate | CategoryDepthError mapped to a clean 422 envelope; unhandled errors fall through to app.ts handler which suppresses raw messages in production |
| T-02-SC | Tampering | npm installs | accept | No package installs — all deps already in apps/api; legitimacy gate handled in plan 02-07 |
</threat_model>
