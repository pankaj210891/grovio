<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → Fastify routes | All incoming HTTP requests are untrusted |
| env vars → Zod parser | Environment config is validated at startup; missing vars crash fast |
| Fastify → PostgreSQL | Internal connection over loopback; credentials via env var |
| Fastify → Redis | Internal connection over loopback; URL via env var |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Fastify error stack traces in production | mitigate | setErrorHandler returns sanitized ApiError in production; raw error logged internally only |
| T-03-02 | Denial of Service | Missing DATABASE_URL or REDIS_URL crashes server | accept | Fail-fast at startup is correct behavior; missing config is a deployment error, not a runtime attack |
| T-03-03 | Tampering | JWT_SECRET min-length enforcement | mitigate | envSchema enforces JWT_SECRET.min(32); any secret shorter than 32 chars is rejected at startup |
| T-03-04 | Spoofing | Health endpoint reveals version info | accept | Version info is not sensitive; health endpoints are standard operational tooling |
| T-03-SC | Tampering | npm installs (fastify, awilix, drizzle-orm, ioredis, pg) | mitigate | slopcheck + blocking checkpoint for [ASSUMED]/[SUS] packages before install |
</threat_model>
