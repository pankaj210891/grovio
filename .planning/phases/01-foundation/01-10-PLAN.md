  <action>
    First: build packages/contracts by running `pnpm --filter @grovio/contracts build` from the monorepo root. This produces packages/contracts/dist/index.js and packages/contracts/dist/money/index.js.

    Second: verify the Metro config's extraNodeModules correctly handles release mode. In release mode, Metro resolves from the package's main/module field (i.e., dist/index.js), not src/. Update apps/mobile/metro.config.js extraNodeModules to point to the packages/contracts directory root (not /src), so Metro uses the package's exports map:
    config.resolver.extraNodeModules["@grovio/contracts"] = path.resolve(monorepoRoot, "packages/contracts");

    This change makes Metro use packages/contracts package.json exports map in both dev and release mode. Metro 0.80+ (shipped with RN 0.83) supports package exports map resolution.

    Third: attempt a local release JavaScript bundle: run `cd apps/mobile && npx expo export --platform android 2>&1 | tail -30` or `npx expo export --platform ios 2>&1 | tail -30`. This exercises Metro's full resolution without requiring a physical device or EAS. The export command performs the same resolution as a release build.

    If the export succeeds (exits 0 or produces a dist/ directory with bundle files), FND-03 is satisfied. If Metro throws a module resolution error for @grovio/contracts, diagnose and fix the extraNodeModules config before proceeding to the checkpoint.

    Common fix if export fails: ensure packages/contracts has been built (dist/ exists), and that the metro.config.js resolver.nodeModulesPaths includes both the app's node_modules and the monorepo root's node_modules.
  </action>
