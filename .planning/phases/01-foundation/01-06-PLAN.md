    private redisKey(flagKey: string): string — returns "ff:" + flagKey.

    async getFlag(key: string): Promise<string | null> — 1) try redis.get(this.redisKey(key)); 2) if hit, return parsed string value; 3) on miss, query db.select().from(featureFlags).where(eq(featureFlags.key, key)).limit(1); 4) if found and isEnabled=true, setex in Redis with TTL = deps.env.FEATURE_FLAG_TTL_SECONDS, return value; 5) if not found, return null.

    async invalidateFlag(key: string): Promise<void> — redis.del(this.redisKey(key)).

    async invalidateAllFlags(): Promise<void> — use redis.keys("ff:*") then redis.del on all keys (use pipeline for efficiency if multiple keys).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → /internal/flags routes | Internal routes should not be publicly exposed; in production, these must be behind network/auth guard |
| Redis cache → application | Redis values are trusted application data; no external input passes through Redis without first going through DB validation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | /internal/flags/* exposed publicly | mitigate | In Phase 6, admin auth middleware is added; for Phase 1 these routes are internal-network only (documented in route handler comment); Phase 1 has no public deployment |
| T-06-02 | Tampering | feature flag value is stored as text (not validated type) | accept | Value flexibility is intentional for Phase 6 admin UI; callers that need typed values must parse/validate at read time |
| T-06-03 | Denial of Service | redis.keys("ff:*") on large key set | accept | Feature flags are a small, bounded set; redis.keys is acceptable for O(N) bounded N |
| T-06-04 | Tampering | Cache poisoning if Redis is compromised | accept | Redis is internal infrastructure; assume network-level protection; no user data flows through flag cache |
</threat_model>
