    Create apps/mobile/metro.config.js using the monorepo-safe approach. Use @expo/metro-config as the base config. The strategy: add watchFolders pointing to the monorepo root packages/ directory, and configure resolver.extraNodeModules to map @grovio/contracts to the packages/contracts directory. This is the most reliable approach for Expo bare + pnpm workspaces.

    Exact metro.config.js structure:
    - const { getDefaultConfig } = require("expo/metro-config");
    - const path = require("path");
    - const projectRoot = __dirname;
    - const monorepoRoot = path.resolve(projectRoot, "../..");
    - const config = getDefaultConfig(projectRoot);
    - config.watchFolders = [monorepoRoot];
    - config.resolver.nodeModulesPaths = [path.resolve(projectRoot, "node_modules"), path.resolve(monorepoRoot, "node_modules")];
    - config.resolver.extraNodeModules = { "@grovio/contracts": path.resolve(monorepoRoot, "packages/contracts/src") }; — Note: this maps to /src directly for dev (no build step needed); plan 01-10 will verify the release build path
    - module.exports = config;

    Note: The extraNodeModules maps directly to packages/contracts/src so Metro can resolve the TypeScript source files directly during development without requiring a build step. This is the standard Expo monorepo approach.
