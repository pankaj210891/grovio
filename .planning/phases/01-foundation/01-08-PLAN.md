    Create root .env.example with the following sections. Every variable must have a descriptive # comment block above it (minimum: purpose line, format line, how-to-obtain line). Use placeholder values that demonstrate the correct format without containing real secrets.

    Section: # =============================================================
    # Grovio — Root Environment Variables
    # Copy this file to .env and fill in your values.
    # These are shared infrastructure variables used by apps/api.
    # Per-app variables are in apps/{app-name}/.env.example
    # =============================================================

    Section: ## Node Environment
    NODE_ENV=development
    (comment: development | test | production — controls logging verbosity and error detail)

    Section: ## Database
    DATABASE_URL=postgresql://grovio:grovio@localhost:5432/grovio
    (comment: PostgreSQL connection string. Format: postgresql://USER:PASSWORD@HOST:PORT/DATABASE. For local dev use docker-compose.yml credentials. For production use a managed Postgres service like Supabase, Railway, or Neon.)

    Section: ## Redis
    REDIS_URL=redis://localhost:6379
    (comment: Redis connection string. Format: redis://HOST:PORT or redis://:PASSWORD@HOST:PORT for authenticated connections. For local dev use docker-compose.yml. For production use Upstash Redis or a managed Redis service.)

    Section: ## Authentication
    JWT_SECRET=replace-with-at-least-32-character-random-secret-string
    (comment: Secret key for HS256 JWT signing and verification. MUST be at least 32 characters. Generate with: openssl rand -hex 32. Never commit your actual secret.)

    Section: ## Feature Flags
    FEATURE_FLAG_TTL_SECONDS=60
    (comment: Redis cache TTL for feature flags in seconds. Controls how quickly a database flag toggle is reflected in API responses. Lower = faster propagation but more DB queries. Default: 60 seconds.)

    Section: ## Google SMTP (Transactional Email)
    GOOGLE_SMTP_USER=your-gmail-address@gmail.com
    GOOGLE_SMTP_PASS=your-gmail-app-password
    (comment for USER: Gmail address used as the SMTP sender for all transactional email. Must be a Google Workspace or Gmail account with SMTP enabled.)
    (comment for PASS: Gmail App Password (not your account password). Generate at: https://myaccount.google.com/apppasswords. Requires 2FA enabled on the Google account.)

    Section: ## OpenSearch
    OPENSEARCH_URL=http://localhost:9200
    (comment: OpenSearch endpoint URL. For local dev use docker-compose.yml. For production use a managed OpenSearch/Elasticsearch service. Used for full-text product search — required from Phase 3 onwards.)

    Section: ## Google Maps / Places
    GOOGLE_MAPS_API_KEY=your-google-maps-api-key
    (comment: Google Maps Platform API key. Required for Google Places autocomplete in checkout and customer address management (Phase 4+). Obtain at: https://console.cloud.google.com → APIs & Services → Credentials. Enable: Maps JavaScript API, Places API.)

    Section: ## Backend Port (apps/api only)
    PORT=3001
    (comment: HTTP port for the Fastify backend API. Default: 3001. Each app also has its own port setting in its per-app .env.example.)

    Do NOT include VITE_* or EXPO_PUBLIC_* vars here — those are per-app vars already in apps/{app}/.env.example files per D-07.
