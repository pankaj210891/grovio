    Steps in order:

    1. Checkout: uses: actions/checkout@v4 with fetch-depth: 0 (needed for Turborepo affected package detection).

    2. Setup pnpm: uses: pnpm/action-setup@v4 with version: "9" (matches pnpm 9.x constraint).

    4. Install dependencies: run: pnpm install --frozen-lockfile (strict: fails if lockfile is out of sync; catches unintentional dependency additions).

    5. Turborepo remote cache login (optional but configured): uses: actions/cache@v4 for .turbo directory with key based on os + turbo.json hash + pnpm-lock hash. This implements local Turborepo cache (no remote token needed for OSS). Comment: "For Vercel remote cache: add TURBO_TOKEN and TURBO_TEAM to GitHub Secrets."

    6. Lint: run: pnpm turbo run lint — fails workflow on any ESLint error.

    7. Typecheck: run: pnpm turbo run typecheck — fails workflow on any TypeScript error.

    8. Format check: run: pnpm turbo run format:check — fails workflow if any file is not Prettier-formatted. Note: this CHECKS, does not write. Developers must run pnpm format:write locally.

    9. Test: run: pnpm turbo run test -- --passWithNoTests — allows packages without tests yet to pass; packages with tests (packages/contracts) must pass their tests.

    10. Build (informational, non-blocking for now): run: pnpm turbo run build -- --continue or handle separately. Actually make build a separate check that DOES fail: run: pnpm turbo run build. This catches import errors that typecheck misses.

    Add environment section: env: CI: "true" (Vitest and other tools respect this).
