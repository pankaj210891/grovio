<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| vendor request → vendor-scoped data | vendorId comes from JWT; services must scope all reads/writes by it (IDOR). |
| invite token → account creation | untrusted token used to mint a vendor_users account. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-15 | Information Disclosure | Banking data exposed via vendor profile read | mitigate | VendorProfileService.getProfile select excludes payout-info columns; banking lives in vendor_payout_info, fetched via a separate owner-only method (D-02). |
| T-06-16 | Elevation of Privilege | Staff invite minting an owner | mitigate | VendorStaffService.invite rejects role='owner'; only manager|staff inviteable (D-05). |
| T-06-17 | Elevation of Privilege | Invite token replay (second accept) | mitigate | accept() rejects when acceptedAt already set or expiresAt passed; token is crypto.randomUUID, 48h expiry. |
| T-06-18 | Tampering | Vendor editing another vendor's inventory/pricing | mitigate | updateInventory/updatePricing verify product.vendorId === request vendorId before write. |
| T-06-19 | Tampering | quantity_reserved manipulation to oversell | mitigate | quantity_reserved is read-only; not writable through vendor methods (D-15). |
| T-06-SC | Tampering | npm installs | mitigate | No installs (argon2, bullmq, drizzle already present). |
</threat_model>
