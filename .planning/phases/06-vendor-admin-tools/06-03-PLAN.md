    Create `apps/api/src/db/migrate-vendor-users.ts` as a standalone tsx script (mirror seed-commerce.ts structure for DB connection + env loading). It runs this idempotent SQL:

    `INSERT INTO vendor_users (id, vendor_id, email, password_hash, role, accepted_at, created_at, updated_at) SELECT gen_random_uuid(), id, email, password_hash, 'owner', NOW(), created_at, NOW() FROM vendors WHERE email IS NOT NULL AND password_hash IS NOT NULL AND NOT EXISTS (SELECT 1 FROM vendor_users vu WHERE vu.vendor_id = vendors.id AND vu.role = 'owner');`

    The `NOT EXISTS` guard makes it idempotent (safe to re-run). Set `accepted_at = NOW()` for migrated owners (they are already active accounts). After insert, log the count of rows inserted and the total owner count.

    Add a package.json script to apps/api: `"db:migrate:vendor-users": "node --use-system-ca --env-file=.env node_modules/tsx/dist/cli.mjs src/db/migrate-vendor-users.ts"`. Run it.
