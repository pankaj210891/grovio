<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| concurrent checkout → wallet | Two tabs may both attempt to spend the same wallet balance. |
| commission split → vendor ledger | Rounding drift in splits would corrupt vendor payouts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Elevation of Privilege | WalletService debit/credit | mitigate | Append-only entries with unique idempotency_key prevent double-spend; cached balance only ever updated alongside a ledger insert in one transaction (WAL-03, Pitfall 3). |
| T-05-07 | Tampering | CommissionService split | mitigate | allocate() (largest-remainder, BIGINT) guarantees splits sum exactly to total — no float multiplication anywhere (MKT-02, Pitfall 1/7). |
| T-05-CR | Tampering | resolveRate cache | accept | Commission rate cached in Redis with TTL; stale rate window is at most one TTL cycle, consistent with feature-flag caching policy. |
</threat_model>
