<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → coupon validation | Client may submit arbitrary/forged coupon codes or discount amounts. |
| return approval → ledgers | Refund + commission reversal must stay consistent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Tampering | coupon discount | mitigate | Discount recomputed server-side from the coupons table; client-provided discount amounts are never used (Pitfall 6). |
| T-05-CPN | Elevation of Privilege | coupon brute force | mitigate | COUPONS_ENABLED gate short-circuits when off; per-code redemption cap enforced; rate-limiting applied at the route layer (05-10). |
| T-05-RFD | Tampering | commission reversal | mitigate | approveReturn inserts a prorated 'reversed' entry via allocate() in the same transaction as the wallet refund; earned rows are immutable (MKT-03, Pitfall 7). |
| T-05-06 | Elevation of Privilege | return ownership | mitigate | createReturnRequest is scoped by customerId; route layer (05-10) enforces requireCustomerAuth so a customer cannot return another customer's order. |
</threat_model>
