<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet → webhook routes | Public unauthenticated webhook endpoints receive provider callbacks. |
| client → basket/checkout/account routes | Authenticated or cookie-bearing customer requests. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Spoofing | webhook routes | mitigate | Signature verified in adapter (constructEvent / HMAC timingSafeEqual) as the first action before any DB write; raw body preserved via plugin-scoped addContentTypeParser (Pitfall 1). |
| T-05-02 | Tampering | webhook idempotency | mitigate | recordWebhookEvent dedup returns 200 on duplicate; finalizeOrder idempotent (PAY-03, Pitfall 8). |
| T-05-06 | Elevation of Privilege | basket/order/wallet/vendor routes | mitigate | requireCustomerAuth on all customer routes (customerId from JWT); vendor routes use vendor JWT guard and ownership checks; basket guest token is httpOnly + random UUID. |
| T-05-CPN | Elevation of Privilege | apply-coupon route | mitigate | Rate-limit hook on /checkout/apply-coupon; COUPONS_ENABLED short-circuit in service. |
| T-05-SC | Tampering | webhook raw-body scope | accept | addContentTypeParser is plugin-scoped; it does not weaken JSON parsing for other routes (verified by full test suite passing). |
</threat_model>
