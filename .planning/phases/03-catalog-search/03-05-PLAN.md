<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| vendor client → catalog API | Untrusted product/variant/image input and JWT cross here |
| vendor → another vendor's resources | IDOR boundary — a vendor must not mutate another vendor's product/image |
| vendor → moderation | A vendor must not self-approve or set status directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-P1 | Spoofing | VendorAuthService.login | mitigate | argon2.verify; identical InvalidCredentialsError for unknown-email and wrong-password (no enumeration); jose HS256 JWT signed with JWT_SECRET (V2) |
| T-03-P2 | Elevation of Privilege | ProductService status | mitigate | status changes only via submit/approve/reject; updateProduct cannot set status except the pending_review→draft reset (Pitfall 3); vendor JWT cannot reach /admin routes |
| T-03-P3 | Tampering | ProductService ownership | mitigate | every owned-product method filters `eq(products.vendorId, vendorId)` — IDOR blocked (V4) |
| T-03-P4 | Tampering | createProduct attributes | mitigate | attribute keys/types validated against attribute_definitions schema registry; Zod parse at route boundary (V5) |
| T-03-P5 | Denial of Service | ImageService oversize upload | mitigate | MAX_IMAGE_SIZE_BYTES checked at presign + ContentLength set on PutObjectCommand so R2/S3 rejects oversize PUT (D-11) |
| T-03-P6 | Tampering | product images orphaning | mitigate | only confirmUpload writes rows; 5-min presign TTL; count guard at presign (Pitfall 4) |
| T-03-P7 | Tampering | bypassing category restriction | mitigate | createProduct checks categories.isRestricted + vendor_category_restrictions at service layer (CAT-06, D-18) |
</threat_model>
