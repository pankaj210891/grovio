## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API routes | All untrusted HTTP input enters here; Zod validation + auth guards are the entry controls |
| vendor JWT → /vendor/* | Token verification boundary |
| admin token → /admin/* | Admin authorization boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-W1 | Spoofing | /vendor/* routes | mitigate | requireVendorAuth preHandler verifies jose JWT and role=vendor before any handler runs (V4) |
| T-03-W2 | Elevation of Privilege | vendor crafting admin token | mitigate | JWT verified with JWT_SECRET; role claim checked; vendor token cannot satisfy admin guard which uses a separate INTERNAL_ADMIN_TOKEN (V4) |
| T-03-W3 | Tampering | request bodies / query params | mitigate | every route .parse()es a @grovio/contracts Zod schema before reaching a service (V5) |
| T-03-W4 | Information Disclosure | error messages | mitigate | domain errors mapped to coded envelopes; unknown errors re-thrown to app.ts handler which suppresses raw messages in production (existing pattern) |
| T-03-W5 | Denial of Service | worker startup blocking HTTP | mitigate | worker started after fastify.listen so HTTP availability is independent of queue health (RESEARCH OQ1) |
</threat_model>
