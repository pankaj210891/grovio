<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| customer client → search API | Untrusted q/filter params cross here |
| product approval → OpenSearch index | Only approved products with only searchable attributes may be projected |
| BullMQ → Redis | Blocking-command connection boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-J1 | Tampering | OpenSearch query injection via q | mitigate | q passed only into structured multi_match value, never raw query-string DSL (V5; Security register) |
| T-03-J2 | Information Disclosure | index document attributes | mitigate | only is_searchable=true attributes projected; dynamic:false on attributes object prevents arbitrary-key leakage (D-15, Anti-Pattern 6, SRCH-04) |
| T-03-J3 | Information Disclosure | non-approved products in search | mitigate | index job only indexes on approval; delete action on revert/archive removes them (D-13, Pitfall 7) |
| T-03-J4 | Denial of Service | BullMQ worker crash on blocking command | mitigate | dedicated ioredis connection with maxRetriesPerRequest:null (Pitfall 1) keeps the worker alive |
| T-03-J5 | Denial of Service | OpenSearch unavailable | mitigate | SearchService.isAvailable() graceful degradation returns empty results instead of unhandled errors |
</threat_model>
