# Plan 11-01: Shadcn UI + Radix UI Foundation Setup

**Phase**: 11 — UX/UI Platform Redesign
**Wave**: 1 (foundational — all later Phase 11 plans depend on this)
**Status**: In Progress

## Goal

Install and configure Shadcn UI (with Radix UI primitives) as the cross-app component library foundation in `packages/ui`. All three web portals (storefront, admin, vendor) consume shared accessible components from `packages/ui`, reducing duplication and establishing a consistent accessible design system aligned with Grovio's existing Tailwind v4 + OKLCH token setup.

## Context

Phase 11 requires a full UX/UI redesign (D-51): a complete cross-app component library in `packages/ui` with accessible primitives. Shadcn UI built on Radix UI provides battle-tested accessible primitives while keeping components fully customizable (code is owned, not npm-locked). This plan wires the foundation before any portal redesign work begins.

## Dependencies

- `packages/ui` already exists with `@theme {}` tokens and tsup build
- All three web apps already import `@grovio/ui/tokens` for CSS
- Tailwind CSS v4 + `@tailwindcss/vite` already configured per app
